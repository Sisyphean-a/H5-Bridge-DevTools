# Iteration 003

## Current Understanding

上一轮唯一未闭环的是浏览器级黑盒证据。实现与单测已经够强，关键是要证明“已打开的第二个页面和第二个面板都会自动看到更新”，并证明图片工具在真实扩展 UI 中能完成两种格式写回。

## Implementation Approach

- 补一个真实 Chromium 扩展端到端脚本，直接加载构建产物而不是只跑 Vitest。
- 为独立打开的 `panel.html` 增加 `tabId` 兜底读取能力，便于在黑盒脚本里稳定复用正式面板逻辑。

## Changes This Iteration

- 新增 `scripts/verify-extension-e2e.mjs`，启动本地测试页面、加载扩展、打开两个同 origin 页面和两个面板页面，并串起实际交互验证。
- 在 `src/panel/panel.tsx` 增加 `?tabId=` 读取逻辑；若当前不是 DevTools host，也能在测试场景下绑定到指定 tab。
- 将 `package.json` 的浏览器验收脚本定为 `npm run test:e2e`，并切换到完整 `playwright` 依赖以加载 Chromium。
